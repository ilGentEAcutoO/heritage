# Claude Code Workflow Skills

Detect intent จากคำพูดของ user แล้วทำตาม skill ที่เหมาะสม

## Global Rules

### 1. ใช้ Skill เสมอเมื่อทำได้

ก่อนทำอะไรให้ตรวจสอบ `.claude/skills/` ว่ามี skill ที่เกี่ยวข้องไหม — ถ้ามี ให้อ่านแล้วทำตาม skill นั้น อย่าทำเองโดยไม่ผ่าน skill เด็ดขาด ถ้าทำงานแล้วพบว่ามี skill ที่ช่วยได้ (เช่น กำลัง commit อยู่แล้วมี git-commit skill) ให้สลับไปใช้ skill ทันที

### 2. สร้างทีม Sub-Agent เสมอ

**อย่าทำงานคนเดียว** — พิจารณาสร้าง sub-agent ทุกครั้งที่:

- งาน parallel ได้ (เช่น implement หลาย tasks พร้อมกัน)
- งานที่ต้องการ cross-review (เช่น agent A implement, agent B test)
- งานที่แยก concern ได้ (เช่น frontend + backend + tests)
- งานที่ต้องการ research หลายแหล่ง

**Model Selection:**

| Role | Model | เมื่อไหร่ |
|------|-------|----------|
| Main Agent (coordinator) | **Opus 4.6** | เสมอ — ตัดสินใจ, ประสานงาน, review |
| Sub-Agent (worker) | **Sonnet 4.6** | งาน implement, test, research ทั่วไป |
| Sub-Agent (critical) | **Opus 4.6** | งาน security review, architecture decisions, complex debugging |

Main agent เป็น Opus 4.6 เสมอ — ทำหน้าที่: แบ่งงาน, ประสานทีม, ตัดสินใจ, review ผลงาน sub-agents, สื่อสารกับ user

### 3. อัปเดต todos.md ทุกครั้ง

อัปเดต `./instruction/work/todos.md` ทุกครั้งที่มีการเปลี่ยนแปลง (timestamp: `YYYY-MM-DD HH:mm`)

### 4. Project-level path เท่านั้น

`./instruction/work/` — ห้ามใช้ `~/.claude/` หรือ user-level path

### 5. Sub-agents ห้ามถาม user

ให้ return กลับ main agent เพื่อตัดสินใจ

### 6. File Lock Registry

เช็คก่อน edit, ปลดล็อคเมื่อเสร็จ — sub-agents ห้ามแก้ไฟล์เดียวกัน

### 7. ห้ามใส่ AI signature

git commits ห้ามมี "Generated by Claude" เด็ดขาด
