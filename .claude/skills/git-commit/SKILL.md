# Git Commit

Professional commits — no AI fingerprints.

## Commit Message Format

```
<type>: <subject>

[optional body]
```

**Subject rules:**
- ≤72 characters
- Imperative mood: "Add", "Fix", "Update" (not "Added", "Fixes")
- No period at end
- Lowercase after type prefix

**Types:**
- `feat:` new feature
- `fix:` bug fix
- `refactor:` code restructure
- `style:` formatting only
- `docs:` documentation
- `test:` adding/updating tests
- `chore:` tooling, deps, config
- `perf:` performance improvement

## Workflow

### 1. Stage changes intelligently

```bash
git status
git diff --stat
```

Group related changes. Split unrelated changes into separate commits.

### 2. Create commit

```bash
git add [files]
git commit -m "feat: add user authentication flow"
```

### 3. If "commit and push"

After commit, trigger **git-push** skill.

## Rules

1. **NEVER** add AI markers: no "Generated by Claude", no "Co-authored-by: Claude"
2. **NEVER** include tool/AI references in commit messages
3. Use conventional commits if project uses them
4. One logical change per commit
5. Write messages as if a human developer wrote them
